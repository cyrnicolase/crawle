DROP LANGUAGE

名称

DROP LANGUAGE -- 移除一个过程语言

大纲

DROP [ PROCEDURAL ] LANGUAGE [ IF EXISTS ] name [ CASCADE | RESTRICT ]

描述

DROP LANGUAGE移除一个之前注册的过程语言 的定义。你必须是一个超级用户或者该语言的拥有者才能使用 DROP LANGUAGE。

注意: 自PostgreSQL 9.1 起，大部分过程语言 已经被做成了"扩展"，因此应该用 DROP EXTENSION而不是 DROP LANGUAGE删除。

参数

IF EXISTS

如果该语言不存在则不要抛出一个错误，而是发出一个提示。

name

一个已有过程语言的名称。为了向前兼容，这个名称可以用单引号包围。

CASCADE

自动删除依赖于该语言的对象（例如该语言中的函数），然后删除所有 依赖于那些对象的对象（见第 5.13 节）。

RESTRICT

如果有任何对象依赖于该语言，则拒绝删除它。这是默认值。

示例

这个命令移除过程语言plsample： 

DROP LANGUAGE plsample;

兼容性

在 SQL 标准中没有DROP LANGUAGE语句。
